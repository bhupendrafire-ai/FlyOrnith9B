# AGENTS.md

## Project Rules

- Treat AgentOrinth as a public repository: every change should be understandable, reviewable, and safe to publish.
- Keep `README.md` beautiful, comprehensive, and accurate for a first-time public reader.
- Update `README.md` in the same commit as every meaningful change that affects setup, usage, architecture, configuration, workflows, safety behavior, or the public project story.
- Commit after every meaningful change; keep commits focused and avoid bundling unrelated work.
- After each commit, write a single-line update comment summarizing the change.
- Preserve existing user config and secrets. Never commit `.env`, virtual environments, dependency folders, local databases, generated workspaces, screenshots, or credentials.
- Before broad code inspection, check the Obsidian second brain for existing AgentOrinth context.
